Motivation

RAG has been around for a while and comes up constantly in conversations. I understood the high-level concept but never dove into the technical implementation details.

Implementation

The basic pipeline works like this: We transform PDF data into "embeddings"—mathematical representations that capture semantic meaning—then store them in Pinecone. When users ask questions, we search the embeddings to find where the answers might be, then feed those results to GPT for a response.

This is an extremely simple RAG implementation. Going forward, I'd like to explore agentic RAG and other techniques—their applications, strengths, and limitations.
