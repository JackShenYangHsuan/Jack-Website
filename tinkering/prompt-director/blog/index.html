Motivation

I often struggle with describing front-end UI details while vibe coding. I could use Figma, but switching tools kills momentum. Words work better, except when you're describing pixel-perfect layouts.

At McKinsey, partners solved this by annotating slides on Zoom while talking through their vision. Real-time markup plus verbal context. I built a tool that brings that workflow to AI prompting.

Implementation

The tech is simple: record audio, capture annotations, then feed both to GPT for detailed prompt generation. I added one twist: stitching before-and-after annotation images together gives the model extra context about what changed.

It's now part of my vibe coding workflow. Whenever I need to describe complicated or precise UI, I spin it up.
