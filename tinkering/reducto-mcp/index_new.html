Motivation

I needed a reliable OCR and document parsing tool for my personal OS. GPT sometimes misses details, especially with handwriting. I learned that Reducto is best in class for this. Rather than calling Reducto's API directly from each workflow, I wanted a standardized MCP interface that any AI agent could use. I also wanted to integrate it with my Gumloop agents so I could call it directly from Slack.

Implementation

You can turn any API into an MCP. The way I think about MCPs is they're basically API calls with a reasoning brain. Instead of structured requests, you interact with MCPs in natural language and they decide how to interface with other services.

This Reducto MCP takes my request in natural language (like "parse this document") and converts it into Reducto API calls. It handles authentication and error handling, then returns the result.

To run this in my Gumloop agent, I had to deploy the MCP server on Vercel so Gumloop could call it. The server is live here: https://reducto-mcp-server.vercel.app/
